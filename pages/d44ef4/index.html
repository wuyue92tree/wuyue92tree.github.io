# 介绍

通过ldap管理nexus3用户

# 版本信息

  • openldap: 2.4.50
  • phpLDAPadmin: 1.2.5
  • nexus3: OSS 3.19.1-01

# nexus3配置

# 新增ldap

创建ldap认证连接，填入ldap服务器相关信息及认证用户信息。点击 verify connection进行连接测试，连接成功的话，点击 save进行用户和用户组的配置

  • Name: ldap名称
  • LDAP server address: ldap连接协议、地址及端口
  • Search base DN: 搜索根节点
  • Authentication method: 认证方式（这里我采用的Simple Authenticaiton）
  • Username or DN: ldap用户dn
  • Password： ldap用户密码
  • Connection rules：连接规则

(opens new window)

# 用户和用户组配置

这里我使用groupOfUniqueNames作为分组，因此模板选择 Generic Ldap Server

# 用户配置

  • Configuration template：配置模板
  • User relative DN：用户目录相对DN（简单说就是在哪个ou下能找到你的用户）
  • User subtree: 基于用户目录查找用户
  • Object class: 用户objectClass
  • User filter：用户搜索过滤条件，比如通过memberOf进行过滤
  • User ID attribute: 用户ID对应ldap的字段
  • Real name attribute：用户RealName对应ldap的字段
  • Email attribute: 用户email对应ldap的字段
  • Password attribute：用户password对应ldap的字段

(opens new window)

# 用户组配置

  • Map LDAP groups as roles：是否启用ldap组和角色映射
  • Group type: 分组类型（动态和静态）
    1. 动态分组
      1. Group member of attribute：用户组过滤属性 (opens new window)
    2. 静态分组
      1. Group realtive DN: 用户组目录相对DN(相对于base DN)
      2. Group subtree：基于用户组目录查找用户组
      3. Group object class: 用户组objectClass
      4. Group ID attribute：用户组ID对应ldap的字段
      5. Group memeber attribute：用户组成员属性对应ldap的字段
      6. Group memeber format：用户组成员format (opens new window)

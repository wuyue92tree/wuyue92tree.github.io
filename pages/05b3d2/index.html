        cdh6.3.2集成tez0.9.2计算引擎

        # CDH集成tez计算引擎

        经测试tez在CDH集群环境中表现一般，经常出现找不到jar包的情况，故将之前的tez业务切换到spark on yarn，该教程仅供参考

        update at: 2020-05-09

        CHD部署相关信息：http://note.youdao.com/noteshare?id=6bfb7bb2805ddedbc3e06816532d480b

        # 软件包

        • maven
        • tez
        • Protobuf-2.5.0

        # 编译环境准备

        # 安装maven

        下载地址: http://mirrors.tuna.tsinghua.edu.cn/apache/maven/maven-3/3.6.3/binaries/apache-maven-3.6.3-bin.tar.gz

        tar -xvf apache-maven-3.6.3-bin.tar.gz
sudo mv apache-maven-3.6.3 /usr/local/maven

# 加入环境变量
export MVN_HOME=/usr/local/maven
export PATH=$MVN_HOME/bin:$PATH

# 验证
mvn -v

        # 将tez部署到hdfs

        cd tez-dist/target
# 从hadoop lib jars中拷贝kryo-2.22.jar到tez文件夹下的lib文件夹下，否则会出现java.lang.ClassNotFoundException: com.esotericsoftware.kryo.Serializer异常
sudo cp /opt/cloudera/parcels/CDH-6.3.2-1.cdh6.3.2.p0.1605554/jars/kryo-2.22.jar tez/tez-0.9.2/lib/

hdfs dfs -mkdir -p /apps/tez-0.9.2

# 在实际测试时，发现上传tez-0.9.2.tar.gz包在执行操作时，会引发yarn异常：org.apache.hadoop.yarn.exceptions.YarnException: Download and unpack failed，提示无法下载解压下面的压缩包，所以我直接将解压后的包put到了hdfs
# hdfs dfs -put tez-0.9.2.tar.gz /apps/tez-0.9.2

hdfs dfs -put tez-0.9.2 /apps/tez-0.9.2/
hdfs dfs -chmod -R 777 /apps/tez-0.9.2/
hdfs dfs -ls /apps/tez-0.9.2/

        # 生成配置文件包

        # 操作目录为`tez-dist/target`

sudo cp -r tez-0.9.2-mimimal tez
sudo mkdir tez/conf
sudo vi tez/conf/tez-site.xml
# tez-site.xml内容===
<configuration>
    <property>
        <name>tez.lib.uris</name>
        <value>${fs.defaultFS}/apps/tez-0.9.2/tez-0.9.2/,${fs.defaultFS}/apps/tez-0.9.2/tez-0.9.2/lib/</value>
    </property>
    <property>
          <name>tez.use.cluster.hadoop-libs</name>
          <value>false</value>
    </property>
</configuration>
# ===================

# tez/lib中包含slf4j的jar包，会打印较多日志，可以在客户端中去掉slf4j-api-1.7.10.jar、slf4j-log4j12-1.7.10.jar这两个jar包，减少日志打印。

# 至此客户端配置完成，将/opt/cloudera/parcels/CDH-6.3.2-1.cdh6.3.2.p0.1605554/lib/tez分发到所有节点，即完成客户端部署。
